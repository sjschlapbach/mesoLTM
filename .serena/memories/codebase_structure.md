# Codebase Structure

`mesoltm` v0.1 is implemented: a mesoscopic (individual-vehicle) Link Transmission
Model ported faithfully from `abmmeso` with added network/routing/control layers.
See `docs/MODEL_CHANGES.md` for the (small) deviations from the reference model.

## Top-level
- `pyproject.toml` — hatchling, PyPI-ready, `requires-python>=3.11`, deps `numpy`+`networkx`; extras `plot`/`ui`/`calib`/`dev`; `mesoltm` console script.
- `LICENSE` — **AGPL-3.0** (was MIT); `NOTICE` — attribution to de Souza et al. + paper DOI.
- `src/mesoltm/` — the package; `examples/` — runnable scripts (outside src); `docs/` — `MODEL_CHANGES.md`. `examples/paper_validation.py` is the canonical **paper-validation** script: one file reproducing the paper's Section 4 figures (Fig 5 lane-drop, 6 W=7 proposed-only, 7 diverge, 8 diverge-random, 9 merge) through the shipped discrete pipeline, with the deliberate divergences (discrete-only/no continuous ref, CFL rejection of Fig 5 dt=6 s, omitted `floor(C·dt)` ablation, out-of-scope signal/real-world Figs 11/12) surfaced on the figures — use it to sanity-check pipeline fidelity after core changes.
- `.github/workflows/` — one workflow per gate: `lint.yml` (pylint), `format.yml` (black --check), `typecheck.yml` (mypy), `test.yml` (pytest, 3.11/3.12 matrix), `release.yml` (build + a commented-out PyPI publish; on default-branch push).
- `CLAUDE.md`, `.ai/` (logs), `.serena/` (config + memories), `.claude/`, `README.md`, `abmmeso/` (temporary reference, to be deleted), `venv/` (git-ignored, 3.11.15).

## Package layout (`src/mesoltm/`)
- `core/` — `base_link`, `link` (discrete LTM, verbatim), `connector_link`, `vehicle` (`Vehicle` carries `route`/`position`/`trajectory` **and `props`** — a free-form mutable metadata dict set at init and updatable any time; the core never touches it, downstream logic/plugins/`color_by` use it; must be JSON-serialisable to survive the history round-trip), `simulation` (`Simulation`: `run()` = batch; also `start()`/`step()`/`current_step`/`total_steps` for step-driven control, and `inject(node_id, vehicle, at_time=None)` for dynamic demand; **per-step history logging** via `record_history`/`history_path`/`history_classify` → `Simulation.history` (a `SimulationHistory`), `save_history(path)`); `core/nodes/` — `base_node` (routing seam), `one_to_one_node`, `diverge_node`, `merge_node`, `general_node_model`, `origin_node` (`add_trip` = injectable demand), `destination_node`.
- `network/` — `network` (builder: add_node/add_link/set_origin/set_destination/compile; auto O/D connectors; capacity-proportional priorities; `compile(injection_budget=N)` sizes connectors for dynamic injection; `compile(record_history=, history_path=, history_classify=)` enables the animation history log), `builders` (grid_network incl. partial, corridor_network, dict I/O), `state` (read-only NetworkState; also `inject(...)` splicing O/D connectors onto a real-link route; `waiting_vehicles(node)` = the live origin entry-queue vehicles, next to `entry_queue(node)` count; `remaining_real_route(vehicle, current_link=None)` = the real links of `vehicle.route` from the current link onward (reads the vehicle's own plan, never recomputes) — shared by `vehicles_in_network` and the recorder).
- `recording.py` (**matplotlib-free**, top-level so `core.simulation` can import it) — per-step animation history: `Frame`/`AgentSnapshot`/`WaitingSnapshot` dataclasses, `capture_frame(state, classify)`. **No route is ever computed at recording time**: each agent's remaining route is read verbatim from `vehicle.route` via `state.remaining_real_route` (the same helper the rerouting interface uses) and logged (`AgentSnapshot.route`/`WaitingSnapshot.route`); the next hop is read back from it. So the log always matches what actually ran, including exogenous mid-run reroutes. Pure node-by-node policy routing stores no plan on the vehicle, so its logged route is empty (honest — the recorder does not fabricate one). Snapshots also carry a copy of the vehicle's `props` metadata (`AgentSnapshot.props`/`WaitingSnapshot.props`), so a `color_by` callable or later analysis can use per-vehicle info; it round-trips through the JSON. In the JSON, each agent/waiting entry is a **keyed object** (field-name keys), not a positional array, so the log is self-describing (`AgentSnapshot(**obj)` rebuilds it); frame occupancy is under `link_occupancy`. `SimulationHistory` (geometry + frames; `save`/`load` JSON), `record_run(sim)` convenience. Snapshots store scalars/lists/dicts only. Exported from `mesoltm`.
- `routing/` — `policy` (RoutingPolicy protocol + StaticRoutePolicy default; `next_link(state: object)` to match the protocol), `shortest_path` (ShortestPathPolicy, networkx; `route(state, a, b)` = full-route planner; `refresh(state)` rebuilds the live graph once so a rerouting plugin can re-plan many vehicles in one step (build once, set `dynamic=False`, reuse); `cost` callback typed `Callable[[int, NetworkState], float]` for congestion-aware costs). `NetworkState` and `VehicleView` are exported from `mesoltm` for external typing.
- `plugins/` (renamed from `control/`) — `plugin`: the general per-step loop hook (run first each step), `Network.compile(..., plugins=[...])`. `Plugin` (base), `FunctionPlugin(fn)`, `ReroutingPlugin(reroute_fn)`. A plugin can change many aspects of the sim; rerouting needs no special hook because each `Vehicle` stores its own `route` and the network only propagates it, so rewriting `vehicle.route` reroutes that vehicle. `ReroutingPlugin` is the simple routing interface: gets `state.vehicles_in_network()` (`VehicleView`s), returns `{vehicle: real_route}` for only the ones to change (applied via `state.set_route`). No step-tracker plugin — the sim loop owns `state.step`.
- `demand/` — `demand.vehicles_from_demand_profile`.
- `metrics/` — `trips` (`collect_trips`, `trip_record`, `summarize_trips`, `write_trips_csv`): per-vehicle `route` driven, `travel_time` = total (incl. access), `access_time` (origin queue + connector), `network_time`, per-link times, keyed by `vehicle_id`. Derived from `Vehicle.trajectory`, which the `Link` fills in `set_inflow`/`set_outflow` (behaviour-preserving; see `MODEL_CHANGES.md` A6).
- `io/` — `scenario` (JSON build/load/save); `cli.py` + `__main__.py` (`python -m mesoltm`).
- `visualizations/` — `plots`: cumulative curves; `plot_link_flow` (sums links into one cut curve) vs `plot_link_flows` (one labelled line per link); `plot_network` (colour links by flow/occupancy/density/capacity, optional `annotate_links` labels, colorbar, parallel links fanned out as arcs); travel-time distribution; per-link travel-time bars; `plot_link_time_series` (one subplot per link: per-vehicle travel time vs entry time as a running average, to show congestion building up). `animation`: agent-movement video/frames from a `SimulationHistory` — `NetworkLayout` (`from_state`/`from_history`, spring-layout fallback; links grouped by **undirected** node pair so opposing/parallel links on one edge fan onto separate arcs — otherwise their agents would coincide; `_PARALLEL_FAN`, plus a small centreline **lane offset** (`_LANE_OFFSET`, `_lane_shift`) so opposing curved arrows also stay apart near the nodes; `min_link` for scale-aware sizing), `render_frame` (discrete FIFO queue slots; **sizes markers/nodes/labels to the network** so dense grids stay readable, with per-link **spacing-aware** marker shrink (`_fit_marker`) so queued agents never overlap; agents drawn as slightly smaller dots; `detail="auto"|"full"|"minimal"` + `show_agent_ids`/`show_next_link`/`show_node_labels` overrides — detail auto-drops when dense but is available on select; **`color_by`** = `"category"` (default) | `"next_link"` (read from the logged `AgentSnapshot.route`) | `None` (uniform, colouring off) | a **callable** `fn(snapshot) -> str` colouring by anything on the snapshot, esp. `snapshot.props`; agents **waiting** to enter (origin queue / source connector) are drawn as a **count badge on the node itself**, not as points, so they are never confused with travelling agents; Okabe-Ito palette; `highlight_links`), `save_frames` (per-step PNGs), `save_animation` (MP4 via ffmpeg / GIF via Pillow fallback; `fps` default 25; `subsample` = frames-per-step playback speed >1 slower / <1 faster; `frames_dir` also dumps PNGs), `animate_history`/`animate_from_history_file` (guides the user if the log is missing)/`animate_simulation`, `expand_frame_indices`, `resolve_palette`. matplotlib imported at module top; the package `__init__` does not import `visualizations`, so `import mesoltm` stays cheap (recording is separate + matplotlib-free).
- `tests/` — **inside the package**, relative imports: `test_regression` (exact match vs abmmeso golden), `test_core`, `test_network`, `test_routing`, `test_scenario`, `test_metrics`, `test_priorities`, `test_injection` (step/inject), `test_rerouting` (ReroutingPlugin), `test_recording` (history logging: remaining-route logged from `vehicle.route`, policy-routed vehicle logs no fabricated plan, log matches a mid-run reroute; matplotlib-free), `test_animation` (renderer + color_by category/next_link/None/callable, detail levels, opposing-link fan, `importorskip("matplotlib")`). `Vehicle.props` + props capture/round-trip covered in `test_core`/`test_recording`. 67 tests total.

## Runtime flow
`Network.compile(...)` → `Simulation` of links + nodes → `.run()` → per-link cumulative
in/outflow arrays + arrived-trip records + per-vehicle trajectories (+ optional CSV via
CLI/scenario or `metrics.write_trips_csv`). Routing is per-vehicle via `vehicle.route`
or a `RoutingPolicy`; plugins run first each step.

Keep this current when files/modules are added, moved, or removed.
