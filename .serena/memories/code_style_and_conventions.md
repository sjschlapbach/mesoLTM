# Code Style & Conventions

- **Python 3.11+** (`requires-python = ">=3.11"`); dev in the 3.11 venv.
- **Packaging:** PEP 621 `pyproject.toml`, **hatchling** backend, **`src/` layout** (`src/mesoltm/`). Import/distribution name `mesoltm`. **Tests live inside the package** (`src/mesoltm/tests/`, relative imports); **examples live outside** `src/` and import the installed package.
- **Runtime deps:** `numpy`, `networkx` (graphs/grids/shortest-path). `matplotlib` is the optional `[plot]` extra. Other extras: `ui` (network editor), `calib` (`scipy`), `dev`.
- **Imports: always at module top level. Never inline (inside functions) and never wrapped in try/except.** `TYPE_CHECKING`-guarded type-only imports are the one allowed conditional form (they avoid circular imports and carry no runtime cost). `matplotlib` is imported at the **top** of `visualizations/plots.py`; `import mesoltm` stays cheap because the package `__init__` does not import `visualizations` (only code that plots pulls in matplotlib). Example scripts import `matplotlib`/`matplotlib.pyplot` at the top and call `matplotlib.use("Agg")` before plotting.
- **Dev deps** (`[dev]`): `pytest`, `pylint`, `black`, `mypy`, `build`.
- **Formatting:** **black** (line length 88). **Lint:** `pylint src/mesoltm examples` (black owns formatting; pylint config in `pyproject.toml` allows the short traffic-engineering names T1/T2/v_f/w/… and disables checks that fight the faithful port). CI lints **both** `src/mesoltm` and `examples`, so example scripts must also score 10.00/10 — common gotchas it enforces: `use-maxsplit-arg` (C0207 — use `str.split(sep, maxsplit=1)` when you only need the first field) and `consider-using-with` (R1732 — wrap `open()` in a `with`). **Types:** `mypy src`, kept clean.
- **Type hints:** on public functions/classes/boundaries. Two-phase-init objects (links/nodes) use concrete zero/empty defaults, not `Optional`, and reset transient per-step scalars to `0` (not `None`) — this keeps mypy clean and is behaviour-identical (see `docs/model/deviations-from-the-paper.md`). **No `# type: ignore[assignment]` in the package** — use a concrete default instead of `None`.
- **Container/return typing (no `Any`):** parametrise generics with **concrete element types** wherever the type is well-defined — homogeneous mappings/lists (`dict[int, Link]`, `list[BaseNode]`, `list[Vehicle]`), tuples of floats (`tuple[float, float]`), known callables (`Callable[..., str]`), matplotlib `Axes`. **Never use `typing.Any`.** For genuinely free-form / heterogeneous values keep a **plain `dict` / `list[dict]`** rather than `Any` or a `TypedDict`: JSON scenario dicts, per-vehicle trip records, `Vehicle.props`/`trajectory`, `SimulationHistory` (de)serialisation payloads, `get_output_records`/`get_arrived_trips` returns. Leave loose, too, the polymorphic-key drawing helpers (`_draw.fan_links`/`draw_nodes` — int link ids live, str from a loaded history), library types (`nx.DiGraph`), and pass-through `**kwargs`. Node-model link-list params are covariant `Sequence[BaseLink]` (read-only) so `list[Link]` callers fit; `Simulation.links: list[Link]` and `NetworkState.links_by_id: dict[int, Link]`. See `.ai/DECISIONS.md` (2026-07-10). Verify with `mypy src` **and** `pyright src/mesoltm` (basic) — both must stay clean.
- **Identifiers (no bare `object`/`Hashable`):** a **link id is `int`** (`BaseLink.link_id: int`). A **node id has no concrete type** — any hashable value passed to `add_node` (`grid_network` uses `(row, col)` tuples, `corridor_network` uses strings) — so it is the named alias **`NodeId` (`= Hashable`)**, defined+justified once in `core/ids.py` and exported as `mesoltm.NodeId`; use `NodeId` everywhere a node id flows, never a raw `Hashable`. In `recording` (ids round-trip through JSON) a link id is **`int | str`** (int live, str after load), a node id stays `NodeId`. `RoutingPolicy.next_link` takes concrete `node: BaseNode`, `state: NetworkState | None`. Only arbitrary `**kwargs: object` and the stringify-anything `_to_key(value: object)` keep `object` — every remaining `object`/`Hashable` must be tied to a deliberately-general interface. See `.ai/DECISIONS.md` (2026-07-10).
- **Time lives only in the `Simulation` loop.** The `Link` stores **no** time state (no `time_step`/`total_time`/`total_steps`/current-step attributes); the loop owns the clock and passes it through arguments (`update_state_variables(t, time_step)`, `get_output_records(sample_time, sim_time_step, total_time)`, `set_inflow/set_outflow(..., step)` with the node forwarding its step index). `start()` uses the step/horizon locally without storing them. Do not re-add time attributes to links (see `docs/model/deviations-from-the-paper.md` A8). **Nodes store only the time they actually read at step time:** `OriginNode` keeps `time_step` (departure test); every other node keeps none — one-to-one/diverge/merge/general have no `start()` override (they inherit the `BaseNode` no-op), and `DestinationNode.start` sizes its arrival array with a local `total_steps`. `Plugin` (user hook) keeps `time_step`/`total_time` as part of its API.
- **Docstrings:** required on every function/class — concise: purpose, `Args`, `Returns`. Inline comments explain *why*, sparingly (not every line).
- **Faithfulness rule:** the ported LTM core (`core/link.py`, node models, 4-phase loop) keeps the reference arithmetic **exactly** — it is intentionally scalar/integer Python, **not** vectorized NumPy (overrides the generic "vectorize hot paths" preference). NumPy is for analysis/output only. `test_regression.py` locks exact equality with `abmmeso`.
- **Variable naming = standard traffic-engineering notation** (not the paper's spellings), values unchanged: `v_f` (free-flow speed), `w` (backward wave speed), `rho_jam` (jam density), `capacity` (= `rho_jam·v_f·w/(v_f+w)`), `density` (ρ), `occupancy` (o). These are the public kwargs (`Link(v_f=…, w=…, rho_jam=…)`) and JSON scenario keys (`"v_f"`, `"rho_jam"`). Map + rationale in `docs/model/deviations-from-the-paper.md` (A5).
- **Merge priorities = shares `alpha_1, alpha_2, …`** (fractions of outbound supply per inbound link, summing to 1). `MergeNode`/`GeneralNodeModel` accept `alpha=` and expose `.alpha`; the discrete nodes still consume the equivalent integer `priority_vector` internally (`core/priorities.py` interconverts). `Network` defaults `alpha` to capacity-proportional; override per node via `Network.set_merge_priorities(node_id, {inbound_link_id: share})`. Behaviour-preserving — see `docs/model/deviations-from-the-paper.md` (B2).
- **License headers:** every source file carries an AGPL-3.0 notice; files with logic ported from `abmmeso`/the paper additionally cite the publication (DOI 10.1016/j.simpat.2025.103088). Keep these headers when adding files.
- Keep `import mesoltm` cheap; export the public API explicitly from `src/mesoltm/__init__.py`.
- File names `snake_case`; classes `PascalCase`.
